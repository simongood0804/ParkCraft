# ParkCraft 项目需求文档

## 1. 项目概述

### 1.1 项目名称
ParkCraft — 停车出库益智游戏

### 1.2 项目简介
ParkCraft 是一款基于网格逻辑的停车出库益智游戏。玩家需要在 N×N 的网格中，通过前后移动多辆堵塞车辆，将目标车辆（长度为 2）从网格唯一出口移出。游戏灵感来源于经典的「华容道」和「Rush Hour」玩法。

### 1.3 项目目标
在 Android 和 iOS 双平台发布一款用户体验流畅、关卡丰富、可扩展性强的停车出库解谜游戏。

## 3. 功能需求

### 3.1 核心玩法

#### 3.1.1 网格系统
- 游戏在 6×6 的方形网格上进行。
- 网格中的每个格子可容纳一辆车的一个单位长度。
- 网格有且仅有一个出口（Exit），出口在右侧边界，行范围为 1~4（不在拐角）。

#### 3.1.2 车辆系统
- **目标车辆（待出库车辆）**：长度为 2，占据网格中连续的 2 个格子。其所在行或列固定，只能沿其朝向（水平或垂直）前后移动。
- **堵塞车辆**：长度为 2 或 3，分布在网格中，阻挡目标车辆的移动路径。
- **所有车辆**：
  - 只能沿自身朝向（水平或垂直）前后移动，不可转向。
  - 不可跨越其他车辆。
  - 不可移出网格边界（出口所在边除外）。
  - 车辆之间不可重叠。

#### 3.1.3 胜利条件
玩家通过移动堵塞车辆，清理出通畅路径，将目标车辆从网格唯一出口完全移出网格，即判定通关。

#### 3.1.4 操作方式
- 玩家点击/拖拽车辆，沿其朝向滑动以移动。
- 移动步数限制在网格范围内，且不可与其他车辆重叠。
- 支持撤销操作（Undo）。

### 3.2 关卡系统

#### 3.2.1 关卡配置文件
关卡通过外部配置文件生成，配置格式推荐 JSON 或 YAML。配置包含以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| `gridSize` | 网格尺寸（固定 6） | 6 |
| `exit` | 出口位置及朝向，行范围 1~4（避免拐角） | `{"row": 2, "col": 5, "orientation": "horizontal"}` |
| `targetCar` | 目标车辆信息 | `{"row": 2, "col": 0, "length": 2, "orientation": "horizontal"}` |
| `blockingCars` | 堵塞车辆列表 | `[{"row": 2, "col": 2, "length": 3, "orientation": "horizontal", "id": "A"}, ...]` |
| `levelId` | 关卡唯一标识 | "level_001" |
| `difficulty` | 难度等级（选填） | "easy" / "medium" / "hard" |

详细格式定义见下文 **第 7 节 — 关卡配置文件规范**。

#### 3.2.2 关卡管理
- 支持快速导入配置文件（本地导入 / 从服务器下载）。
- 关卡按难度分组展示。
- 支持关卡解锁机制：通关当前关卡后解锁下一关。
- 内置至少 12 个初始关卡，难度按最少通关步数划分：

| 难度 | 步数范围 | 关卡数量 |
|------|----------|----------|
| 初阶 | 15~20 步 | 4 关 |
| 中阶 | 21~29 步 | 4 关 |
| 高阶 | 30~44 步 | 4 关 |

### 3.3 游戏功能

#### 3.3.1 基本功能
- 开始游戏、暂停、继续。
- 步数计数（记录玩家移动车辆的次数）。
- 计时器（记录通关用时）。
- 撤销上一步操作（支持连续撤销到初始状态）。
- 重开当前关卡（重置到初始布局）。
- 提示系统（为玩家提供下一步移动建议）。

#### 3.3.2 关卡选择界面
- 以网格或列表形式展示所有关卡。
- 区分锁定/解锁状态、是否已通关。
- 显示每个关卡的最佳步数和最佳用时。

#### 3.3.3 通关反馈
- 通关动画（目标车辆驶出出口）。
- 通关弹窗：展示通关步数、用时、是否打破最佳记录。
- 进入下一关的快捷入口。

### 3.4 设置与个性化
- 音效开关（背景音乐、移动音效、通关音效）。
- 震动反馈开关。
- 语言设置（至少支持中文和英文）。
- 游戏数据重置（清除通关记录）。

## 4. UI/UX 设计需求

### 4.1 设计规范
- 遵循 **Material Design** 设计规范。
- 在 Android 和 iOS 上保持一致的视觉风格和交互体验。
- 适配不同屏幕尺寸和分辨率（手机和平板）。

### 4.2 界面结构
| 界面 | 说明 |
|------|------|
| 启动页 | 应用 Logo、加载动画 |
| 主菜单页 | 开始游戏、关卡选择、设置入口 |
| 关卡选择页 | 关卡列表/网格视图，显示状态和最佳成绩 |
| 游戏页 | 顶部信息栏（步数、用时）、中部游戏网格、底部操作按钮（撤销、提示、暂停） |
| 设置页 | 音效、震动、语言、数据重置 |
| 通关弹窗 | 步数、用时、最佳记录、下一关按钮 |

### 4.3 视觉要求
- 车辆使用鲜明的颜色区分不同类型（目标车辆用醒目颜色如红色，堵塞车辆用不同颜色）。
- 出口处有明显标识（箭头或高亮）。
- 车辆移动动画流畅（平滑滑动过渡）。
- 网格线清晰，背景简洁。

### 4.4 无障碍设计
- 车辆和按钮有足够大的点击区域（建议最小 44×44 pt）。
- 关键操作支持读屏软件标签。

## 5. 非功能需求

### 5.1 性能
- 游戏运行帧率 ≥ 60 FPS。
- 应用启动时间 ≤ 3 秒。
- 关卡配置加载时间 ≤ 1 秒。

### 5.2 兼容性
- iOS：支持 iOS 13.0 及以上。
- Android：支持 Android 7.0 (API 24) 及以上。
- 适配主流屏幕尺寸（包括刘海屏、挖孔屏）。

### 5.3 存储
- 关卡配置存储在应用沙盒内。
- 游戏进度（通关记录、最佳成绩）使用本地数据库（如 SQLite / Hive）持久化。
- 关卡配置文件解析失败时需有优雅的降级处理。

### 5.4 可扩展性
- 关卡配置采用标准化格式，支持第三方工具生成关卡。
- 代码架构分层清晰（数据层、业务逻辑层、UI层），便于后续增加新功能（如在线关卡市场、每日挑战等）。

## 6. 数据模型

### 6.1 车辆模型
```dart
enum Orientation { horizontal, vertical }

class Car {
  final String id;           // 车辆唯一标识
  int row;                   // 起始行（0-indexed）
  int col;                   // 起始列（0-indexed）
  final int length;          // 车身长度（2 或 3）
  final Orientation orientation; // 朝向
  bool isTarget;             // 是否为目标车辆
}
```

### 6.2 关卡模型
```dart
class Level {
  final String levelId;       // 关卡唯一标识
  final int gridSize;         // 网格尺寸
  final Exit exit;            // 出口
  final Car targetCar;        // 目标车辆
  final List<Car> blockingCars; // 堵塞车辆列表
  final String difficulty;    // 难度等级（选填）
}
```

### 6.3 出口模型
```dart
class Exit {
  final int row;             // 出口所在行
  final int col;             // 出口所在列
  final Orientation orientation; // 出口朝向（决定目标车辆驶出方向）
}
```

## 7. 关卡配置文件规范

### 7.1 JSON 格式示例
```json
{
  "levelId": "level_001",
  "difficulty": "easy",
  "gridSize": 6,
  "exit": {
    "row": 2,
    "col": 5,
    "orientation": "horizontal"
  },
  "targetCar": {
    "id": "T",
    "row": 2,
    "col": 0,
    "length": 2,
    "orientation": "horizontal"
  },
  "blockingCars": [
    { "id": "A", "row": 0, "col": 0, "length": 3, "orientation": "horizontal" },
    { "id": "B", "row": 0, "col": 3, "length": 2, "orientation": "vertical" },
    { "id": "C", "row": 2, "col": 2, "length": 3, "orientation": "horizontal" },
    { "id": "D", "row": 2, "col": 3, "length": 2, "orientation": "vertical" },
    { "id": "E", "row": 4, "col": 1, "length": 3, "orientation": "horizontal" }
  ]
}
```

### 7.2 说明
- `exit.row` 必须在 1~4 范围内（避免拐角），`exit.col` 固定为 5（网格右侧边界）。
- `exit.orientation` 固定为 `horizontal`。
- 目标车辆（`targetCar`）的长度固定为 **2**，朝向固定为 `horizontal`。
- 堵塞车辆（`blockingCars`）的长度为 **2** 或 **3**。
- `row` 和 `col` 均为从 0 开始的索引，范围 `0 ≤ row, col < 6`。
- 所有车辆初始位置不得相互重叠，且不能超出网格边界。

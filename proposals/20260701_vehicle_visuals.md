# 车辆视觉优化 —— 从色块替换为车型图像

> **状态**：`草稿`
>
> **提案人**：项目团队
> **创建日期**：2026-07-01
> **最后更新**：2026-07-01

---

## 1. 背景与动机

当前游戏中所有车辆均以纯色圆角矩形色块表示，缺乏车型辨识度和视觉趣味性。虽然颜色可以区分不同车辆，但玩家无法直观感受到"这是一辆车"，游戏沉浸感不足。

## 2. 目标

- 将矩形色块替换为具有车型特征的精致图像
- 长度 2 与长度 3 的车辆使用不同车型，视觉上有明显差异
- 目标车辆使用警车（黑白配色），与其他车辆形成鲜明区别
- 所有车型保持统一的卡通/扁平画风

## 3. 方案设计

### 3.1 核心思路

使用 Flutter `CustomPainter` 在 Canvas 上绘制车辆图像（自顶向下的俯视图），保留现有的碰撞检测和拖拽逻辑不变，仅替换 `GridPainter._drawCars()` 中的绘制实现。

**不引入图片资源**：全部使用代码绘制矢量图形，确保：
- 无额外资源文件，安装包不增大
- 任意分辨率下清晰
- 易于后续调色和适配

### 3.2 详细设计

#### 车辆类型映射

| 长度 | 车型 | 画风描述 |
|------|------|----------|
| 3（长车） | 公交车 | 长方体车身 + 车窗线条 |
| 3 | 半挂卡车 | 车头 + 货柜，拖钩 |
| 3 | 水泥搅拌车 | 车头 + 圆形搅拌罐 |
| 3 | 危险品运输车 | 车头 + 货柜 + 危险品标志 |
| 2（短车） | 跑车 | 流线型车身 + 尾翼 |
| 2 | 出租车 | 轿车 + 车顶灯箱 |
| 2 | 救护车 | 轿车 + 车顶警灯 + 十字标志 |
| 2 | 小轿车 | 标准轿车轮廓 |
| 2（目标） | **警车** | 黑白配色 + 车顶警灯 + "POLICE" |

#### 车型与颜色对应

保留现有的颜色分配逻辑，但让车型决定基本轮廓，颜色决定涂装：

```
水平长车（3格）：[车头][车厢][车厢]  — 侧面视图
垂直长车（3格）：[车头]  ← 俯视图，车头朝下
                 [车厢]
                 [车厢]
```

#### 绘制实现

在 `lib/widgets/game_grid.dart` 的 `GridPainter` 中新增 `_drawVehicle()` 方法，替代现有的纯色矩形绘制：

```dart
class GridPainter extends CustomPainter {
  // ... 现有字段不变

  void _drawVehicle(Canvas canvas, Car car, Rect rect) {
    final paint = Paint()..color = _getCarColor(car);
    
    switch (car.type) {  // Car 模型需新增 type 字段
      case VehicleType.bus:
        _drawBus(canvas, rect, paint);
        break;
      case VehicleType.truck:
        _drawTruck(canvas, rect, paint);
        break;
      // ...
    }
  }
}
```

#### 模型层调整

`Car` 模型新增 `VehicleType` 枚举字段：

```dart
enum VehicleType {
  // 长度 3
  bus,          // 公交车
  semiTruck,    // 半挂卡车
  cementTruck,  // 水泥搅拌车
  hazmatTruck,  // 危险品运输车
  // 长度 2
  sportsCar,    // 跑车
  taxi,         // 出租车
  ambulance,    // 救护车
  sedan,        // 小轿车
  // 目标车（长度 2）
  policeCar,    // 警车（黑白）
}
```

车辆类型分配逻辑：在 `LevelParser` 解析时，根据车辆长度和索引从预设列表中分配车型。

## 6. 备注

- 车型图片全部使用 Canvas API 绘制，无需外部图片资源
- 绘制复杂度可逐步迭代：先实现扁平的"车形轮廓"，后续再细化车窗、车灯等细节
- 如果画风需要调整，可集中修改 `_drawVehicle` 系列方法
